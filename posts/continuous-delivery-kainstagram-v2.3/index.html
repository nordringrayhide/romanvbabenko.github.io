Consistence of whole system can be checked about 15 minutes. Great!

Why I created this?

I was looking for a job, and I wanted to show my skills in a user friendly way.

Is the code perfect?

All declined questions start with ‘it depends on’

Perfect code can be achieved through refactoring and as an experienced programmer I can speed up the process. The very first step is to make it work, which is the primary goal of any code. The secondary goal is to write beautiful code that is easy to maintain.

Let me show you

Example #1

Full rebuilding of VM took about 10 minutes and at first it satisfied me, and I was happy. After I rebuilt the VM ten times, I got bored, and started thinking about using caching. Constantly downloading the same packages takes too much time and vagrant-cachier plugin helped me speed up the process. What was the result?

before: 8:25 min
after: 3:38 min

That is a 60% improvement! Amazing! Bottleneck was removed. Naturally I could use additional improvements such as incremental rollbacks and interim images, but at that moment rebuilding time satisfied me, so I postponed them.

Example #2

Gusteau provisioner included an old version of the Serverspec testing framework. I couldn’t use Guard and new matchers with old version, so I decided to upgrade. If I had decided to stay with old version, I have developed matches on my own and run tests by hand. I upgraded Serverspec.

Thereby I got a new and up-to-date set of matchers and removed manually test runtime losses.

I am not a fan of total upgrading, because logically, if debugging is the process of removing bugs, then coding is a process of creating them. In other words, be careful with your desire to get something new, because you simply don’t know how many drawbacks an upgrade will introduce. The only thing which can help you is testing. I had already tested, so I upgraded.
