Dropbox as Remote Git Hosting

2016/04/30

You really do not have to use a lorry to move a sack of cement. Wheelbarrow fits better for this.

Let me show you my git storage evolution:

  • Github (private reps 12$ per month)
  • Gitlab - VPS(Linode, 2GB, 20$ per month)
  • Gogs - VPS(Linode, 1GB, 10$ per month) - love it so much, gonna do another post about it
  • finally, git-remote-dropbox - free

As you can see I sacrificed features I did not need. My main aim was my own remote git storage: ui, collaborative things, webhooks did not matter at all. If it’s vital for you, then try Redmine, it has all the things I’ve mentioned.

Tips

Note: You do not have installed any desktop Dropbox clients It better exclude repo folder from sync with Dropbox Never touch reps directly on Dropbox

## Installation

You must have installed Python and Pip

pip install git-remote-dropbox

Register App

Let’s go to Dropbox app console and register an app.

Tips

  • Create “App folder” type of app
  • Don’t you use the word “dropbox” in the name of your app?
  • Development mode is enough for private git storage.
  • After you have done that, generate OAuth 2 token and create credentials file in your home directory
~/.git-remote-dropbox.json

{
    "token": "xxxxxxxx"
}

Create repository

prepare folder

~% mkdir git-remote-dropbox
~% cd git-remote-dropbox

init repo

~/git-remote-dropbox% vi README.md
~/git-remote-dropbox% git init

commit first changes

~/git-remote-dropbox[master]% git add .
~/git-remote-dropbox[master]% git ci -m"first commit"

push changes to dropbox

~/git-remote-dropbox[master]% git remote add origin dropbox://git/git-remote-dropbox
~/git-remote-dropbox[master]% git push origin master

Now you can login into your Dropbox account and find the repo in /Apps/remote-dbo-x/git/git-remote-dropbox

Clone repository

git clone dropbox://git/git-remote-dropbox

Tips

Check out your branch after you’ve cloned a repo. git checkout master

## Improvements

Let’s add a bit sugar into our candy

function cloneRemoteDropbox() { git clone dropbox://git/$1 && \
  cd $1 && git checkout master }
function initRemoteDropbox() { mkdir $1 && cd $1 && git init && \
 git remote add origin dropbox://git/$1 && \
 echo "# $1" > README.md && git add . }

alias git-remote-dx-clone='cloneRemoteDropbox'
alias git-remote-dx-init='initRemoteDropbox'

So now you can do that

clone repo named foo

git-remote-dx-init foo

push foo into Dropbox

git commit -m"first commit"
git push origin master

and clone foo repo at any time at any place

git-remote-dx-clone foo

Actually, that’s all.

Recap

This article is about the way you can pass to understand that what you really need. Sometimes big and complex apps are overhead.

An approach like that lowers your expenses and save your time and efforts. There are a lot of things that need your attention more, than useless features at the moment.
